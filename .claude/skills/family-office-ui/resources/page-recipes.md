# Page recipes

## Shared home shell
The home/dashboard should not be a KPI wall.
Recommended structure:
1. Opening statement area with date, family greeting, and one meaningful highlight
2. Large featured module based on recency or importance
3. Three mode gateways: Family, Operations, Advisory
4. Recent activity rendered as an elegant timeline or digest, not a raw audit log

## Family landing
Use when the page is about people, photos, or family history.

Recommended structure:
1. Editorial header with featured image or collage
2. Story block or featured moment
3. Profile strip or family roster
4. Memory collections / albums / milestones
5. Upcoming events or notes

Visual emphasis:
- image scale
- captions
- layered modules
- warm tone

## Operations landing
Use for property or household management.

Recommended structure:
1. Spatial overview: map, property selector, or hero image
2. Priority tasks and issues
3. Schedule / maintenance / vendor stream
4. Financial snapshot only as supporting information
5. Document drawer or records section

Visual emphasis:
- operational clarity
- map/list/table harmony
- strong section labels
- no photo-album styling

## Advisory landing
Use for trusts, guidance, planning, and finance summaries.

Recommended structure:
1. Quiet executive summary
2. Key figures in restrained stat blocks
3. Recommendations or recent advisories
4. Documents and meeting notes
5. Timeline of upcoming decisions / deadlines

Visual emphasis:
- memo and briefing feel
- disciplined spacing
- understated authority

## Photo gallery page
Avoid a pure thumbnail dump.
Use one of these patterns:
- featured story image + supporting grid
- event journal with date/location notes
- cinematic strip with expandable detail

## Family profile page
Recommended structure:
1. Portrait / hero / monogram block
2. Bio and role in family
3. Current updates or recent activity
4. Linked collections: albums, homes, travel, documents
5. Relationship or lineage context where appropriate

## Property detail page
Recommended structure:
1. Full-width property hero or map
2. Key facts rail
3. Current tasks and open issues
4. Vendors and contacts
5. Expense/documents/history below

## Trust / document page
Recommended structure:
1. Title and status line
2. One-paragraph summary in plain English
3. Key sections or clauses overview
4. Advisors and ownership/responsibility
5. Related files, dates, and actions
